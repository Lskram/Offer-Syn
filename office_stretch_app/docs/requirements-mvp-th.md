# Requirement MVP แอปแจ้งเตือนยืดเส้นสำหรับคนทำงานหน้าคอม

## 1. เป้าหมายของผลิตภัณฑ์

แอปนี้มีเป้าหมายเพื่อช่วยผู้ใช้ที่นั่งทำงานหน้าคอมเป็นเวลานานให้พัก ยืดเส้น และปรับพฤติกรรมระหว่างวันอย่างสม่ำเสมอ เพื่อลดอาการปวดเมื่อยสะสมจากการทำงานลักษณะ office syndrome

แนวคิดของ MVP คือทำให้ผู้ใช้ได้รับคุณค่าหลัก 3 อย่างก่อน:

- รู้ว่าควรยืดส่วนไหน
- ได้โปรแกรมท่าที่สอดคล้องกับอาการหลัก
- ได้ตัวช่วยเตือนและเริ่มทำท่าอย่างรวดเร็ว

## 2. กลุ่มผู้ใช้

- คนทำงานออฟฟิศ
- นักเรียน นักศึกษา
- ฟรีแลนซ์
- คนที่ใช้คอมพิวเตอร์เป็นเวลานาน

ช่วงอายุเป้าหมาย: 20-59 ปี

## 3. ขอบเขต MVP

### In Scope

- แบบสอบถามตอนเข้าใช้ครั้งแรก
- ระบบเลือกโปรแกรมท่าบริหารจากอาการ
- หน้า Home แสดงโปรแกรมวันนี้และเวลารอบเตือน
- หน้ารอบการทำท่าแบบมี timer
- คลังท่าบริหาร
- Tips และหลักการดูแลตัวเอง
- Settings สำหรับปรับช่วงเวลาและรอบเตือน
- ประวัติการทำท่าระดับพื้นฐาน

### Out of Scope

- Login / Register / Google Login
- Backend และ cloud sync
- Firebase Cloud Messaging
- Profile รูปผู้ใช้
- ระบบแชตหรือ AI coach

## 4. ขอบเขตเนื้อหาใน MVP

รอบแรกให้รองรับเฉพาะกลุ่มอาการที่มีข้อมูลท่าชัดเจนจากเอกสารต้นทาง:

- คอ บ่า ไหล่
- สะบัก และหลังส่วนบน
- หลังส่วนล่าง และเอว

หมวดที่ยังไม่รวมใน MVP:

- ข้อมือ
- ตา

เหตุผล:
ข้อมูลท่าบริหารสำหรับสองหมวดนี้ยังไม่ครบในชุดข้อมูลปัจจุบัน จึงยังไม่ควรเปิดเป็น recommendation หลักจนกว่าจะตรวจสอบเนื้อหาเพิ่มเติม

## 5. แบบสอบถามเริ่มต้น

คำถามที่ใช้ใน MVP:

1. บริเวณที่ปวดบ่อยที่สุด
2. ระดับความปวด
3. ใช้คอมพิวเตอร์วันละกี่ชั่วโมง
4. ปกติยืดเส้นหรือออกกำลังกายบ่อยแค่ไหน

เหตุผลที่ต้องเพิ่มคำถามระดับความปวด:
ข้อมูลโปรแกรมท่าที่มีอยู่ผูกกับความถี่แจ้งเตือนโดยตรง

- ปวดมาก -> ทุก 30 นาที
- ปวดปานกลาง -> ทุก 45 นาที
- ปวดน้อย / ป้องกัน -> ทุก 60 นาที

## 6. Logic การแนะนำโปรแกรม

ใช้ rule-based recommendation ใน MVP

สูตรเริ่มต้น:

- `pain_area + pain_level` -> เลือกชุดท่าหลัก
- `pain_level` -> กำหนดรอบแจ้งเตือนเริ่มต้น
- ผู้ใช้สามารถ override รอบแจ้งเตือนใน Settings ได้

ตัวอย่าง:

- คอ บ่า ไหล่ + ปวดมาก -> โปรแกรมคอเร่งด่วน + เตือนทุก 30 นาที
- หลังส่วนล่าง + ปวดปานกลาง -> โปรแกรมเอวคลายตึง + เตือนทุก 45 นาที

## 7. หน้าจอหลักของ MVP

2. Home

หมายเหตุ:

## 8. Functional Requirements

### 8.1 Questionnaire

- ผู้ใช้ต้องตอบแบบสอบถามก่อนเข้าหน้า Home ครั้งแรก
- เมื่อกดสร้างโปรแกรม ระบบต้องเลือกโปรแกรมที่เหมาะสมและตั้ง reminder interval เริ่มต้นให้

### 8.2 Home

- แสดงชื่อโปรแกรมปัจจุบัน
- แสดงจำนวนท่าและเวลารอบถัดไป
- มีปุ่มเริ่มทำท่า
- มีปุ่มเลื่อนการเตือน
- แสดงประวัติล่าสุด

### 8.3 Exercise Session

- แสดงชื่อท่า คำอธิบาย และเหตุผลของท่า
- มี countdown timer
- เมื่อครบเวลาให้ไปท่าถัดไปอัตโนมัติ
- ผู้ใช้กดทำครบ ข้าม หรือเลื่อนเวลาได้

### 8.4 Exercise Library

- แสดงโปรแกรมแยกตามจุดปวด
- แสดงความต่างของแต่ละระดับอาการ
- ผู้ใช้สามารถเริ่มโปรแกรมจากคลังได้โดยตรง

### 8.5 Tips

- แสดงคำแนะนำเชิงยศาสตร์
- แสดงแนวทางการพักระหว่างงาน
- มีข้อความเตือนด้านความปลอดภัย

### 8.6 Settings

- เปิด / ปิดการแจ้งเตือน
- เปิด / ปิดเสียงแจ้งเตือน
- ตั้งรอบแจ้งเตือน เช่น 30 / 45 / 60 / 90 / 120 นาที
- ตั้งช่วงเวลาที่อนุญาตให้เตือน เช่น 08:00-16:30
- เริ่มทำแบบสอบถามใหม่ได้

## 9. Non-Functional Requirements

- ใช้งานง่ายระหว่างวัน
- เน้นกดเริ่มท่าได้เร็ว
- โทนสีสบายตา
- ทำงานได้แม้ไม่มีอินเทอร์เน็ต
- ควรเปิดแอปครั้งแรกแล้วเข้าใจ flow ได้ภายในไม่กี่นาที

## 10. สถาปัตยกรรม MVP

### Mobile

- Flutter

### State

- Local state ภายในแอป

### Storage

- ใช้ local persistence เพื่อเก็บ questionnaire / settings / history / next reminder
- หากข้อมูลซับซ้อนขึ้นค่อยย้ายไป local database

### Notification

- ใช้ local notifications เป็นแนวทางหลัก
- ไม่ใช้ FCM ใน MVP
- schedule ตาม active window และ reminder interval ของผู้ใช้

## 12. ข้อควรระวังด้านสุขภาพ

ต้องมี disclaimer ในแอปว่า:

- แอปนี้เป็นเครื่องมือดูแลตนเองเบื้องต้น
- ไม่ใช่การวินิจฉัยหรือการรักษาโรค
- หากมีอาการปวดรุนแรง ชา ร้าว หรือเวียนหัว ให้หยุดและพบผู้เชี่ยวชาญ
